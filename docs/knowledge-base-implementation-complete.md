## 🏗️ System Architecture

### Core Components
```
scripts/knowledge-base/
├── fetch-issues.js              # GitHub API data extraction
├── categorize-issues.js         # AI-powered issue categorization  
├── generate-docs.js             # Documentation generation
├── advanced-analysis.js         # AI insights with temporal weighting
├── reality-check-analysis.js    # Bias detection and validation
├── issue-trends-visualization.js # Visual trend analysis
└── README.md                   # System documentation
```

### Data Flow
1. **Fetch** → Raw GitHub issues with full metadata
2. **Categorize** → AI-powered classification into problem categories
3. **Generate** → Structured markdown documentation with navigation
4. **Analyze** → Advanced insights with environmental correlation
5. **Validate** → Reality check analysis detecting bias artifacts
6. **Visualize** → ASCII charts showing contributor patterns

### Output Structure
```
docs/knowledge-base-generated/
├── README.md                    # Main overview and navigation
├── {category}/                  # Category subdirectories
│   └── {category}.md           # Category-specific documentation
├── generation-summary.json     # Metadata and statistics
├── reality-check-analysis.json # Validation results
└── issue-trends-analysis.json  # Trend data and patterns
```
